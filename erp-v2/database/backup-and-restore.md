# 紘嘉 ERP 備份與復原手冊

## 每月本機備份

1. 使用系統管理員登入 ERP。
2. 開啟「系統管理 → 備份與版本」。
3. 按「下載完整資料備份」。
4. 將 JSON 檔存入公司加密磁碟，勿放在私人雲端、通訊軟體或共用下載資料夾。
5. 回到同一畫面按「驗證備份檔」，確認格式與各資料表筆數一致。

瀏覽器基於安全限制，不能在未經使用者操作時自動把檔案寫入電腦，因此系統會提供上次備份日期與每月提醒，由管理員按一次完成下載。

## 私人附件

完整 JSON 備份包含資料庫紀錄與私人附件路徑，但不包含 Supabase Storage 中的實際檔案。`hr-private` 私人檔案區應依 Supabase 管理政策另行匯出，並與資料備份存放在同一個月份資料夾。

## 每季復原演練

1. 新增一個與正式環境分離的測試 Supabase 專案。
2. 依順序執行 `schema.sql` 及所有 `migration-*.sql`。
3. 先在 ERP「備份與版本」驗證 JSON 備份。
4. 由資料庫管理員依外鍵順序匯入測試環境，不可直接覆蓋正式專案。
5. 抽查員工、案場、班表、打卡、薪資、現金領取與網站通知筆數。
6. 使用測試帳號登入 ERP 與員工端，確認權限隔離及打卡 RPC 正常。
7. 記錄演練日期、備份檔名、驗證人及發現問題。

## 發生事故時

- 立即停止會擴大異常的新增、刪除或批次匯入操作。
- 保存錯誤畫面、時間、操作人及最近一次有效備份。
- 優先在測試專案還原並核對資料，確認無誤後才安排正式復原。
- 不要直接在正式資料庫執行來源不明或未核對版本的 SQL。
